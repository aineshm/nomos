# Nomos web viewer (deck.gl + Mapbox)

The pitch, on the real San Francisco map: the **trained** coordination policy
(full-opacity WheeledLab cars, flowing smoothly, near-zero crashes) overlaid on
the **untrained** "today's traffic" shadow world (faint, gridlocked). The delta
between them *is* the RL result — no separate traffic sim.

## Run it (3 steps)

```bash
# 1. Export trajectories (trained + untrained) -> public/trajectories.json
python -m smoothride.demo.export_web

# 2. (optional) real SF basemap: copy the config and paste a Mapbox token
cp smoothride/demo/web/config.example.js smoothride/demo/web/config.js
#   -> edit config.js, or just append ?token=pk.xxx to the URL

# 3. Serve the repo root and open the viewer
python -m http.server 8000
#   open http://localhost:8000/smoothride/demo/web/
```

Without a Mapbox token it still runs — you lose the satellite/street basemap but
keep the road overlay on a dark background.

## What's on screen

- **Cars** — WheeledLab-style RC-car meshes (`carmesh.js`), colored by speed
  (red = stopped → green = fast), red when crashed. Trained = solid, untrained =
  faint shadow.
- **Trails** — recent motion paths (deck.gl `TripsLayer`), split at respawns.
- **Roads** — the real OSM street segments for downtown SF.
- **HUD** — live per-world counters: moving / jammed / crashed / trips done.
- **Controls** — play-pause, scrub, world toggle (both / trained / untrained),
  trails & roads toggles.

## Files

| file | role |
|---|---|
| `index.html` | shell: loads deck.gl + mapbox-gl (CDN), HUD, controls |
| `app.js` | viewer logic: layers, animation loop, HUD, fallback path |
| `carmesh.js` | procedural WheeledLab RC-car + pedestrian meshes |
| `style.css` | layout / theme |
| `config.example.js` | Mapbox token template (copy to `config.js`) |
| `public/trajectories.json` | generated by `export_web.py` |
| `models/` | drop a real `.glb` here to swap the procedural car |

## Tuning (top of `app.js`)

`HEADING_SIGN`, `HEADING_OFFSET_DEG` (fix car facing), `CAR_SIZE`, `TRAIL_LEN`,
`CAR_GLB` (path to a real asset).

## Bigger scenes

`export_web.py` defaults to the clean 24-car downtown delta. For a denser city
export, point `--trained/--untrained` at the city checkpoints and raise
`--agents` (use `--stride 2` to keep the JSON small):

```bash
python -m smoothride.demo.export_web --trained runs/trained_city.msgpack \
  --untrained runs/untrained_city.msgpack --agents 200 --peds 40 --stride 2
```

> Note: the city checkpoints were trained on the larger SF bbox; the default
> export uses the downtown graph, so for a faithful city export wire the matching
> map/bbox into `export_web.py` (see `render_zoom.py --downtown` for the pattern).

## Cinematic upgrade — real WheeledLab physics

This web path is the always-works demo. The rigid-body **WheeledLab/Isaac**
render (trained setpoints → frozen low-level controller → RTX render) is scoped
in `../isaac/` and runs on a GPU box.
